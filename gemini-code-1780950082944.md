# وثيقة الإجراءات الفنية ونظام ربط الـ WebApp بالبوت (Membership Automation Architecture)

## 1. دورة حياة إجراءات العضوية والاشتراكات
1. **الطلب الأولي:** يفتح العضو واجهة الـ HTML المصغرة (Telegram WebApp) عبر الشات الخاص بالبوت.
2. **اختيار الباقة ومعالجة البيانات:** يقوم العضو بإدخال بياناته واختيار الباقة (مثل باقة السنة بتوفير 1800 جنيه). عند الضغط على زر التفعيل، ترسل دالة `Telegram.WebApp.sendData()` مصفوفة بيانات موحدة من نوع JSON إلى البوت بصورة مشفرة وآمنة.
3. **الدفع والمراجعة:** يستقبل البوت بيانات الـ JSON في الخاص ويقوم بإرسال رسالة تأكيدية للعضو تحتوي على طرق الدفع المتوفرة (مثل فوري أو محفظة إلكترونية).
4. **التفعيل التلقائي والتنبيهات:** بمجرد تأكيد الإدارة لعملية الدفع، يقوم البوت بتوليد رابط دعوة مخصص وصالح للاستخدام لمرة واحدة فقط للجروب المشترك `CreateChatInviteLink`.
5. **متابعة المدد الطرد الآلي:** يسجل البوت تاريخ البداية والنهاية في قاعدة البيانات (مثل Supabase أو Firebase). وقبل انتهاء الاشتراك بـ 48 ساعة، يرسل البوت تنبيهاً آلياً للتجديد في الخاص. وإذا لم يقم العضو بالتجديد، يستدعي البوت دالة الـ API الخاصة بالتلجرام `BanChatMember` متبوعة بـ `UnbanChatMember` ليتم إخراجه تلقائياً وفورياً من الجروب حفاظاً على تنظيم المنصة.

## 2. معايير الأمن وحظر الاختراق (Security Hardening)
* جميع البيانات المتبادلة تمر عبر واجهة برمجة تطبيقات التلجرام الرسمية المشفرة بطبقة حماية تمنع التلاعب بالأسعار من طرف العميل (Client-Side Manipulation Validation).
* يتم التحقق من صحة توقيع البيانات (Hash Validation) في جانب الخادم (Backend) عبر تشفير مفتاح الـ Bot Token للتأكد من أن البيانات صادرة بالفعل من الـ WebApp المعتمد الخاص بك وليس من طرف خارجي.
